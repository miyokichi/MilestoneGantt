# Milestone Gantt

依存関係付きのマイルストーン・ガントチャート。**Flask + Flask-SocketIO** によるリアルタイム多人数編集と、外部スクリプト／AIエージェント向けの **REST API** を備えています。データは SQLite (`gantt.db`) に自動保存されます。

```
◆  Milestone Gantt
   日 / 週 / 月のズーム · ドラッグ操作 · 依存関係 · コメント · REST API
```

## 主な機能

- **ガントチャート表示** — 日 / 週 / 月の3段階ズーム。今日のラインを常時表示。
- **マイルストーン** — 名前・期限・レーン・担当者・進捗（リング表示）・重要度（★1〜3）・カラーを設定可能。
- **依存関係** — マイルストーン同士を矢印で接続。
- **コメント** — 各マイルストーンにスレッド形式でコメントを記録。
- **レーン / カテゴリ** — レーンをカテゴリでグルーピング。**ドラッグで並べ替え**・カテゴリへ移動、折りたたみ可能。
- **リアルタイム同期** — 複数ブラウザで同時編集。変更は即座に全員へ反映。
- **REST API** — `curl` や `fetch`、AIエージェントからマイルストーン等を操作可能。API経由の変更もブラウザへリアルタイム反映。
- **一括追加 / JSONインポート・エクスポート** — テキスト一括登録とバックアップ。
- **SQLite自動保存** — 初回起動時にサンプルデータを投入。

## 起動

```bash
python app.py
```

起動後、ブラウザで **http://localhost:5000** を開きます。

```
==================================================
  ◆  Milestone Gantt — 依存関係付き
==================================================
  → http://localhost:5000
==================================================
```

> 初回起動時、`gantt.db` が無ければ自動作成され、サンプルのカテゴリ・レーン・マイルストーンが投入されます。

## 使い方（UI）

画面は左の **レーンパネル** と右の **タイムライン（ガント）** で構成されます。

### マイルストーンを追加する

追加方法は3通りあります。

1. **「＋ 追加」ボタン**（左上）
   モーダルが開き、名前・期限日・レーン・担当者・進捗・重要度・カラーを入力して保存します。
   - **期限日** は日付ピッカーのほか、`月/B`（10日）・`月/M`（20日）・`月/E`（末日相当=30日）のプリセットからも選べます。
   - **重要度** は ★（低）〜★★★（高）。ズームによって表示/非表示が変わります（後述）。

2. **グリッドを右クリック**（おすすめ）
   タイムライン上の **レーンの空きスペースを右クリック** → 「**＋ ここにマイルストーンを追加**」を選ぶと、**右クリックした位置のレーンと日付** が自動で入力された状態で追加モーダルが開きます。

3. **一括追加**（[後述](#一括追加)）

### マイルストーンを編集・移動する

- **クリック** … 編集モーダルを開く。
- **左右にドラッグ** … 期限日を変更（ドラッグ中、日付が即時に追従）。
- **右クリック** … コンテキストメニュー（コメントを見る / 編集 / 依存関係を引く / 削除）。
- マーカー上の **リング** は進捗、**中央の数字** は日付、右上の **小さな丸** は重要度を表します。

### 依存関係を引く

1. 左上の **「依存関係を引く」** ボタンを押してリンクモードに入る（またはマーカー右クリック →「依存関係を引く」）。
2. **接続元** のマーカーをクリック → **接続先** のマーカーをクリックすると矢印が引かれます。
3. `Esc` でキャンセル。
4. 既存の矢印を **クリック** すると削除できます。

### コメントを書く

各マイルストーンにスレッド形式でコメントを残せます。

1. マーカーを **右クリック → 「コメントを見る」** で右側に詳細ドロワーが開きます。
2. 下部の入力欄に名前（任意）と本文を入力し、**送信**（または `Ctrl + Enter`）。
3. コメントにカーソルを合わせると表示される `✕` で個別削除。
4. `Esc` でドロワーを閉じます。

### レーンとカテゴリ

左パネルでレーンとカテゴリを管理します。

- **レーン追加** … パネル下部の「＋ レーン追加」。
- **カテゴリ追加** … 「＋ カテゴリー追加」。レーンをグループ化できます。
- **名前変更** … レーン名／カテゴリ名を **クリック** してその場で編集。
- **並べ替え（ドラッグ）** … レーン行を **ドラッグ** して上下に並べ替え。
  - 行の **上半分／下半分** にドロップすると挿入位置（前／後）が青い線で表示されます。
  - **カテゴリ見出しにドロップ** すると、そのカテゴリへレーンを移動します。
- **並べ替え（ボタン）** … レーン行ホバー時の `↑` `↓` ボタンでも移動可能。
- **カテゴリ割り当て** … レーン行の `⊞` ボタンからカテゴリを選択／未分類に変更。
- **折りたたみ** … カテゴリ見出しの `▼` でそのカテゴリのレーンを開閉。
- **削除** … `✕` ボタン（レーン削除時はそのレーンのマイルストーンも削除されます）。

### パネル幅の調整

左パネルとタイムラインの **境界線をドラッグ** するとパネル幅を変えられます（120〜480px）。幅はブラウザに記憶され、次回も復元されます。

### ズームと重要度フィルタ

- 左上の **日 / 週 / 月** ボタンで時間軸の粒度を切り替えます。
- ズームごとに表示される重要度の下限が変わります（情報過多を防ぐため）。
  - **日** … すべて表示（★以上）
  - **週** … ★★以上
  - **月** … ★★★のみ
- 週・月表示時はヘッダーに現在の表示下限（「★★以上」など）が出ます。

### 一括追加

「**≡ 一括追加**」から、1行1件のテキストでまとめて登録できます。

```
名前 | 日付 | レーン | 担当者 | 進捗% | 重要度(1-3)
```

- 区切りは `|` のほか、カンマ・タブも可。`#` で始まる行はスキップ。
- 日付は `2026-05-01` / `2026/5/1` / `5/15`（今年）/ `+7`（今日から7日後）に対応。
- 入力中にプレビューとエラーチェックが表示されます。

例：
```
設計完了 | 2026-05-01 | Phase 1 | Airi | 100
API実装 | +14 | Phase 2 | Bunta
テスト完了 | +28 | Phase 3
```

### エクスポート / インポート

- **↓ JSON** … 現在の全データを `gantt_export.json` としてダウンロード。
- **↑ JSON** … エクスポートしたJSONを読み込んで全データを置き換え。

## REST API

`curl`・`fetch`・任意のスクリプトや **AIエージェント** からマイルストーンを操作できます。
**API経由の変更も、接続中のブラウザへリアルタイムに反映** されます。

- ベースURL: `http://localhost:5000/api`
- リクエストボディ・レスポンスはいずれも JSON（`Content-Type: application/json`）
- CORS有効（`Access-Control-Allow-Origin: *`）
- 認証なし（ローカル利用前提）

### エンドポイント一覧

| メソッド | パス | 説明 |
|---|---|---|
| `GET` | `/api` | エンドポイント一覧を返す |
| `GET` | `/api/state` | 全データ（categories / lanes / milestones / deps / comments） |
| `GET` | `/api/lanes` | レーン一覧 |
| `GET` | `/api/milestones` | マイルストーン一覧 |
| `POST` | `/api/milestones` | 作成 |
| `GET` | `/api/milestones/<id>` | 1件取得（コメント込み） |
| `PATCH` | `/api/milestones/<id>` | 部分更新 |
| `DELETE` | `/api/milestones/<id>` | 削除（依存関係も除去） |
| `GET` | `/api/milestones/<id>/comments` | コメント一覧 |
| `POST` | `/api/milestones/<id>/comments` | コメント追加 |
| `DELETE` | `/api/milestones/<id>/comments/<cid>` | コメント削除 |
| `GET` | `/api/deps` | 依存関係一覧 |
| `POST` | `/api/deps` | 依存関係を追加 |
| `DELETE` | `/api/deps/<id>` | 依存関係を削除 |

### マイルストーンのフィールド

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| `name` | string | 作成時推奨 | 名前（空なら `(無名)`） |
| `date` | string | **必須** | 期限日 `YYYY-MM-DD` |
| `lane` | string | 任意 | レーン名（既定 `—`） |
| `assignee` | string | 任意 | 担当者 |
| `progress` | int | 任意 | 進捗 0–100（範囲外は丸め） |
| `rank` | int | 任意 | 重要度 1–3（既定 2） |
| `color` | string | 任意 | 例 `#7c9885`（未指定はランダム） |

### 使用例

```bash
# 1) まず現在の盤面を取得（レーンID・マイルストーンIDを把握）
curl http://localhost:5000/api/state

# 2) マイルストーンを作成
curl -X POST http://localhost:5000/api/milestones \
  -H "Content-Type: application/json" \
  -d '{"name":"設計レビュー","date":"2026-07-15","lane":"Phase 1","progress":0,"rank":2}'
# → {"ok": true, "milestone": {"id": "...", ...}}

# 3) 進捗を更新（idは作成時のレスポンスから）
curl -X PATCH http://localhost:5000/api/milestones/<id> \
  -H "Content-Type: application/json" \
  -d '{"progress":100}'

# 4) コメントを追加
curl -X POST http://localhost:5000/api/milestones/<id>/comments \
  -H "Content-Type: application/json" \
  -d '{"text":"レビュー指摘を反映しました","author":"agent"}'

# 5) 依存関係を追加（A → B）
curl -X POST http://localhost:5000/api/deps \
  -H "Content-Type: application/json" \
  -d '{"from_id":"<A-id>","to_id":"<B-id>"}'

# 6) 削除
curl -X DELETE http://localhost:5000/api/milestones/<id>
```

### Python からの例

```python
import requests

BASE = "http://localhost:5000/api"

# 盤面を読む
state = requests.get(f"{BASE}/state").json()
lane = state["lanes"][0]["name"]

# 作成
ms = requests.post(f"{BASE}/milestones", json={
    "name": "本番リリース", "date": "2026-08-01", "lane": lane, "rank": 3,
}).json()["milestone"]

# 更新
requests.patch(f"{BASE}/milestones/{ms['id']}", json={"progress": 50})
```

> **AIエージェントから使う場合**: まず `GET /api/state` でレーンとマイルストーンの一覧・IDを取得し、それを使って作成・更新・コメントを呼び出します。エラー時は `{"ok": false, "error": "..."}` と適切なHTTPステータス（400 / 404）が返ります。

## データの保存場所

- すべてのデータは `app.py` と同じディレクトリの **`gantt.db`**（SQLite）に保存されます。
- リセットしたい場合は `gantt.db` を削除して再起動すると、サンプルデータで初期化されます（削除前に **↓ JSON** でバックアップ推奨）。
